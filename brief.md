# Client Brief — Relay

*I'm treating myself like a client. This is the problem I'm paying to have solved.*

**Who I am.** I run a two-person digital studio with my cofounder. Between us we carry roughly eight active client accounts — websites, builds, and a steady drip of "hey can you just…" requests.

**What's broken.** Client requests arrive from everywhere — DMs, email, voice notes, calls. There's no single place they live, so my cofounder and I constantly double-cover or drop work, because neither of us can see the full board across every client at once. Worse: about 80% of what comes in is routine — copy tweaks, content swaps, small page changes — work that doesn't need my brain but still eats my day. By the time I've triaged it, I've burned the hours I was trying to save.

**What I've tried.** Slack — turns into a firehose, nothing stays tracked. Notion and Linear — clean, but they're just databases: my AI can't actually *do* the work from them, and clients won't log in to a project tool. A Trello board per client — eight boards, no shared brain.

**What I need.** One shared workspace — Relay — that does three things together that no tool I've tried does. **One:** my cofounder and I run every client account from a single shared board — real project and task management, so nothing slips between us. **Two:** clients drop requests straight onto that board in plain language — no login, no tool to learn. **Three:** an AI connected to the same files actually *completes* the work — not tracks it, not summarizes it, *does it* — and writes the completion note when it's done. Every tool I've tried organizes the work and hands it back to me. Relay finishes it. And because the files are the source of truth, my team, my clients, and the AI all edit the same thing — and I own every deliverable outright, so the work survives even if the app dies.
