# pattern — sort-before-you-work

**The move:** Never touch a request until it has a lane. Sorting is a separate, deliberate step from doing.

**Why:** A wrong completion is worse than no completion. The lane is the seatbelt — it goes on first, every time.

**Looks like:**
> Request in → restate in one line → assign 🟢/🟡/🔴 + reason → *then* act.

**Failure it prevents:** diving into work that turns out to be irreversible, ambiguous, or out of scope — discovered only after damage.
