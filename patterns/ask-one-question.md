# pattern — ask-one-question

**The move:** When a request is unclear, write exactly **one** specific question and stop. Don't guess; don't ask five things.

**Why:** A guess on live client work is expensive. A vague "can you clarify?" is annoying and bounces the work back unhelpfully. One sharp question gets the answer in one reply — the request resumes the moment it lands.

**Looks like:**
> 🟡 HOLD → "Where's the new team photo — in the folder, or can you send it?"

**Anti-pattern:** a checklist of five questions ("which photo? what size? where? when? any caption?"), or the lazy "please provide more detail." Both make the client do Relay's thinking.

**The one-question test:** could a single answer unblock the whole request? If yes → ask it. If it would take several answers, the request is probably really *several requests* → see `split-the-bundle`.

**Composes with:** `sort-before-you-work` (HOLD is a sort outcome) · `split-the-bundle` (a bundle often hides the one real ambiguity).

**Seen in:** `studio/clients/northwind/req-015` — "swap the team photo for the new one" named no asset. One question ("which photo, where?"), then it cleared the moment Dana dropped the file.
