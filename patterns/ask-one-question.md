# pattern — ask-one-question

**The move:** When a request is unclear, write exactly **one** specific question and stop. Don't guess; don't ask five things.

**Why:** A guess on live client work is expensive. A vague "can you clarify?" is annoying. One sharp question gets the answer in one reply.

**Looks like:**
> 🟡 HOLD → "Where's the new team photo — in the folder, or can you send it?"

**Failure it prevents:** both the confident-wrong guess and the lazy "please provide more detail" that bounces the work back unhelpfully.
