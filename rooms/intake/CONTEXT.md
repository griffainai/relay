# ROOM — Intake  (Layer 2)
_You're here when a new request arrives._

## What happens in this room
Raw, plain-language input becomes clean, sorted request files. The client writes however they want; you do the structuring.

## What good looks like
1. **Preserve the client's exact words** under "Request (as filed)."
2. **Split bundles** — one ask = one file (`req-NNN.md`). Dana sends 3 asks in a sentence; that's 3 files.
3. **Restate** each ask in one line, in your own words, as the title.
4. **Sort** each into a lane (→ `reference/lane-protocol.md`).
5. Never drop an ask to ambiguity — unclear → 🟡 HOLD with one question.

## Exit
Every ask from the message exists as a file with a lane and a reason. Then walk to the next room: Delivery (🟢), Escalations (🔴), or wait (🟡).

## Anti-patterns
- Paraphrasing the client's original words. (Keep them verbatim.)
- Merging three asks into one file because they came in one message.
- Guessing an interpretation to avoid a HOLD.
