# ROOM — Delivery  (Layer 2)
_You're here to complete a 🟢 CLEAR request._

## What happens in this room
You turn a sorted request into a finished deliverable + a completion note. This is the room where "the work gets done."

## What good looks like
1. Load the client's `CONTEXT.md` (voice + constraints) — always.
2. Pick the **one** matching skill from `skills/` (copy-edit, content-update, page-change, seo-meta).
3. Produce the deliverable into `clients/<client>/deliverables/`.
4. Stay inside the request — **no scope creep**, no "while I was in there."
5. Write the 3-line completion note (`reference/completion-note-template.md`).
6. Update the request file status → `complete`; update the client `STATE.md`.

## Exit
Deliverable exists, note written, STATE updated. The folder now matches reality.

## Anti-patterns
- "I also improved a few other things." → invented scope.
- Marking complete when you only did part. → never.
- Skipping the client's voice rules.
