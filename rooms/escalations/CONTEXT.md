# ROOM — Escalations  (Layer 2)
_You're here to hand a 🔴 request up to a human._

## What happens in this room
You do the *thinking* so the operator only has to do the *judgment*. A great escalation is almost a completed task minus the one human decision.

## What good looks like
1. Name the **hard trigger** that fired (irreversible / money / production / outbound / scope / relationship / novelty).
2. **Gather the context** the operator would otherwise have to dig up (where the draft is, what's blocking, what's already done).
3. End with a **recommendation** — the fastest path to done.
4. Route to the client's owner (`team/people.md`); set status → `escalated`.
5. Use `reference/escalation-note-template.md`.

## Exit
The operator can act in under a minute without asking you anything.

## Anti-patterns
- "This needs you, not sure." → no trigger, no context, no recommendation.
- Attempting the work anyway "to be helpful." → never on a hard trigger.
- Quietly downgrading to CLEAR to keep the board clean. → forbidden.
